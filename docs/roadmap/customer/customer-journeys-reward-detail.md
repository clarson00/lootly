## Monetary Value Display

### Should We Show $ Value?

| Pros | Cons |
|------|------|
| Clear value proposition | May feel transactional |
| Helps decision-making | Values fluctuate |
| Competitive advantage | Some rewards hard to value |
| Transparency | May cheapen the experience |

### Recommendation: Show Approximate Value

Display monetary value with "~" prefix to indicate approximation:

```
ğŸ’° ~$12 value        (single item)
ğŸ’° ~$35+ value       (multiple items, use +)
ğŸ’° Priceless! ğŸ†     (badges, experiences)
```

### Value Calculation

```typescript
interface RewardValueDisplay {
  // For product rewards
  monetaryValue?: number;           // $12.00
  displayValue: string;             // "~$12 value"

  // For points
  pointsValue: number;              // 200 pts

  // For badges (no monetary value)
  isBadge: boolean;
  badgeRarity?: 'common' | 'rare' | 'legendary';
}

function formatRewardValue(reward: Reward): string {
  if (reward.type === 'badge') {
    return 'ğŸ† Badge';
  }

  if (reward.monetaryValue) {
    if (reward.hasMultipleItems) {
      return `~$${reward.monetaryValue}+ value`;
    }
    return `~$${reward.monetaryValue} value`;
  }

  return `ğŸª™ ${reward.pointsValue} pts`;
}
```

## Auto-Removal Logic

### What Gets Auto-Removed

| Type | Removal Trigger | Notes |
|------|-----------------|-------|
| **Time-bound promos** | End date passed | "Double points this weekend" |
| **Expired voyages** | Deadline passed + failed | Voyage with deadline |
| **One-time rewards** | Already earned | "First visit bonus" |
| **Limited quantity** | Sold out | "First 100 customers" |
| **Seasonal** | Season ended | "Holiday special" |

### What Stays (Doesn't Auto-Remove)

| Type | Behavior |
|------|----------|
| **Repeating rewards** | Reset and stay visible |
| **Always-available** | Never removed |
| **Voyages in progress** | Stay until complete or abandoned |
| **Failed voyages** | Move to "Try Again" section |

### Database Schema

```sql
-- Track hidden rewards per customer
CREATE TABLE customer_hidden_rewards (
  id TEXT PRIMARY KEY,
  customer_id TEXT NOT NULL,
  business_id TEXT NOT NULL,
  reward_id TEXT,              -- NULL if hiding a rule
  rule_id TEXT,                -- NULL if hiding a reward
  ruleset_id TEXT,             -- NULL if not a voyage
  hidden_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(customer_id, reward_id),
  UNIQUE(customer_id, rule_id),
  UNIQUE(customer_id, ruleset_id)
);

CREATE INDEX idx_hidden_customer ON customer_hidden_rewards(customer_id);
```

### API

```typescript
// Hide a reward/rule
POST /api/customer/rewards/:id/hide
POST /api/customer/rules/:id/hide
POST /api/customer/voyages/:id/hide

// Unhide
DELETE /api/customer/rewards/:id/hide
DELETE /api/customer/rules/:id/hide
DELETE /api/customer/voyages/:id/hide

// Get hidden items
GET /api/customer/hidden
// Response: { rewards: [...], rules: [...], voyages: [...] }
```

### Database Schema

```sql
-- Customer ratings for rewards/voyages
CREATE TABLE reward_ratings (
  id TEXT PRIMARY KEY,
  customer_id TEXT NOT NULL,
  business_id TEXT NOT NULL,
  reward_id TEXT,                  -- NULL if rating a voyage
  ruleset_id TEXT,                 -- NULL if rating a reward
  rating INT NOT NULL,             -- 1-5 stars
  worth_it TEXT,                   -- 'yes', 'okay', 'no'
  feedback_text TEXT,
  -- Voyage-specific ratings
  difficulty_rating INT,           -- 1-5 scale
  time_rating INT,                 -- 1-5 scale
  fun_rating INT,                  -- 1-5 scale
  would_repeat TEXT,               -- 'definitely', 'maybe', 'no'
  -- Abandonment feedback
  abandonment_reasons TEXT[],      -- Array of reasons
  rated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(customer_id, reward_id),
  UNIQUE(customer_id, ruleset_id)
);

-- Participation analytics (aggregated)
CREATE TABLE reward_analytics (
  id TEXT PRIMARY KEY,
  business_id TEXT NOT NULL,
  reward_id TEXT,
  ruleset_id TEXT,
  period_start DATE NOT NULL,
  period_end DATE NOT NULL,
  views INT DEFAULT 0,
  starts INT DEFAULT 0,            -- For voyages
  completions INT DEFAULT 0,
  redemptions INT DEFAULT 0,
  hides INT DEFAULT 0,
  avg_rating DECIMAL(3,2),
  worth_it_yes INT DEFAULT 0,
  worth_it_okay INT DEFAULT 0,
  worth_it_no INT DEFAULT 0,
  avg_time_to_complete INTERVAL,
  updated_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(business_id, reward_id, period_start),
  UNIQUE(business_id, ruleset_id, period_start)
);

-- Voyage drop-off tracking
CREATE TABLE voyage_dropoff_analytics (
  id TEXT PRIMARY KEY,
  business_id TEXT NOT NULL,
  ruleset_id TEXT NOT NULL,
  step_number INT NOT NULL,
  period_start DATE NOT NULL,
  dropoffs INT DEFAULT 0,
  total_reached INT DEFAULT 0,
  dropoff_rate DECIMAL(5,2),
  common_reasons TEXT[],
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_ratings_business ON reward_ratings(business_id);
CREATE INDEX idx_ratings_reward ON reward_ratings(reward_id);
CREATE INDEX idx_analytics_business ON reward_analytics(business_id, period_start);
```
